### Ключевые технические решения

| Решение | Обоснование |
|---|---|
| Vite + React + TS | Быстрая сборка, строгая типизация, стандарт индустрии |
| MUI | Готовая дизайн-система для дашбордов, таблиц, карточек |
| React Query (TanStack Query) | Кэширование, пагинация, retry, мутации «из коробки» |

| FSD (Feature-Sliced Design) | Масштабируемая архитектура с чёткими границами слоёв |
| MSW | Мок-сервер уровня Service Worker — не влияет на код приложения |

## Этап 1 — Мок-данные и API-слой

**Цель:** Сгенерировать offline-датасет (5000 записей), типизировать API-ответы, написать MSW-хендлеры.

### Чеклист

- [x] Установить `@faker-js/faker`, реализовать offline-генератор с `faker.seed(42)` — 5000 персон
- [x] Сгенерировать связанные сущности: `family`, `education`, `employment`, `housing`
- [x] Описать типы `PersonSummary` (для списка) и `PersonDetails` (с вложенными связями) — намеренно разделены
- [x] Создать `shared/api/client.ts` (инстанс axios/fetch с base URL)
- [x] Создать `shared/api/persons.api.ts` (функции `getPersons`, `getPersonById`, `updatePerson`)
- [x] Создать `shared/api/stats.api.ts` (функция `getStats`)
- [x] Написать MSW-хендлеры: `GET /persons` (пагинация, поиск, фильтр, сортировка на сервере)
- [x] Написать MSW-хендлер: `GET /persons/:id`
- [x] Написать MSW-хендлер: `GET /stats` (агрегаты считаются в хендлере, не на фронте)
- [x] Добавить искусственные задержки (100–800ms) и ~5% случайных ошибок (404/500)

### Ключевые технические решения

| Решение | Обоснование |
|---|---|
| `faker.seed(42)` | Детерминированная генерация — консистентные данные при каждом запуске |
| Разделение `PersonSummary` / `PersonDetails` | Как в реальных REST API — список не тащит вложенные связи |
| Серверная пагинация, поиск, фильтр, сортировка | Имитация реального бэкенда, валидация подхода до написания UI |
| Агрегаты в MSW-хендлере | Фронт не делает вычислений — только отображает |
| Случайные ошибки + задержки | Проверка обработки ошибок, лоадеров и retry на раннем этапе |

### Критерий «готово»

MSW запускается, `GET /persons?page=1&pageSize=20&search=…` возвращает корректные данные, `GET /persons/:id` — карточку, `GET /stats` — агрегаты. В панели Network DevTools видны ответы мок-сервера.

### Ключевые технические решения

### Ключевые технические решения

| Решение | Обоснование |
|---|---|
| React Hook Form + Zod per tab | Валидация на уровне каждого таба; общая схема — на уровне формы |
| `useMutation` + query invalidation | Мутация автоматически сбрасывает кэш списка — консистентность без ручного refetch |
| Режим чтения/редактирования | Пользователь не может случайно изменить данные; явное действие «редактировать» |
